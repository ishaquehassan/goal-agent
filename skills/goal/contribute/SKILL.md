# /goal:contribute

You are a contribution assistant. Detect what the user learned or fixed during this session, classify it, and submit a PR to the goal-agent repo.

## When This Runs

This skill is triggered in TWO ways:
1. User manually runs `/goal:contribute`
2. **HARD RULE:** At the END of every session, before saying goodbye, the system MUST ask: "Kuch seekha is session mein jo plugin mein contribute karna chahoge?" and if user says yes, run this flow.

## Step 1: Detect Changes

Compare the local plugin files against the remote repo to find what changed:

```bash
# Get the plugin directory
PLUGIN_DIR="$HOME/.claude/plugins/data/goal-agent@ishaquehassan"

# If local dev repo exists, use that instead
DEV_DIR="$(pwd)"
if [ -f "$DEV_DIR/version.json" ] && grep -q "goal-agent" "$DEV_DIR/.claude-plugin/plugin.json" 2>/dev/null; then
  PLUGIN_DIR="$DEV_DIR"
fi

# Check for uncommitted changes
cd "$PLUGIN_DIR"
git diff --name-only
git diff --staged --name-only
```

**Files to check for universal improvements:**
- `CLAUDE.md` (browser automation rules, commenting rules, key rules)
- `skills/goal/*/SKILL.md` (any skill definition changes)
- `natural-content-rules.md` (content writing rules)
- `agents/*.md` (agent definitions)
- `dashboard/**` (dashboard code)
- `install.sh`, `install.ps1`, `update.sh`, `update.ps1` (installation scripts)

**Files that are ALWAYS user-specific (NEVER contribute):**
- `goal-definition.md`, `goal-profile.md`, `strategy-roadmap.md`
- `progress-tracker.md`, `content-calendar.md`, `contacts-network.md`
- `engagement-log.md`, `profile-audit.md`, `blockers.md`
- `research-findings.md`
- Any file in the state directory that contains personal goal data

## Step 2: Classify Changes

For each changed file, determine:

| Classification | Examples | Action |
|---------------|----------|--------|
| **Universal fix** | Lightbox prevention rewrite, comment flow fix, search strategy | Include in PR |
| **New hard rule** | "Emojis mandatory", "Never scrollIntoView" | Include in PR |
| **Behaviour change** | Content quality rules, variation rules | Include in PR |
| **User-specific** | Goal data, contacts, engagement logs | SKIP, never include |
| **Mixed** | CLAUDE.md has both universal rules and user's goal data | Extract only universal parts |

**CRITICAL:** CLAUDE.md contains BOTH universal rules (browser automation, commenting rules) AND user-specific data (contacts, progress, LinkedIn profile). When contributing CLAUDE.md changes:
- ONLY include changes to: Browser Automation section, Commenting/Reply Rules section, Key Rules section, DigitalHire Rules, Event Verification Rule
- NEVER include changes to: Current Progress, Key Contacts Status, GitHub Profile, LinkedIn Profile, StackOverflow, Speaking Events, Content Strategy sections

## Step 3: Show Changes to User

Present a summary:

```
Session Learnings Detected:
===========================

Universal improvements (will be in PR):
1. [FILE] CLAUDE.md - Comment posting workflow rewritten (find+ref method)
2. [FILE] skills/goal/engage/SKILL.md - Lightbox prevention updated
3. [FILE] natural-content-rules.md - Emoji rule changed to mandatory

Skipped (user-specific):
- engagement-log.md (personal data)
- contacts-network.md (personal data)

Want me to create a PR with these improvements?
```

Wait for user confirmation before proceeding.

## Step 4: Create PR

```bash
cd "$PLUGIN_DIR"

# Ensure we're on main and up to date
git checkout main
git pull origin main

# Create feature branch
BRANCH_NAME="improve/session-$(date +%Y%m%d-%H%M%S)"
git checkout -b "$BRANCH_NAME"

# Stage only universal files
git add [files determined in Step 2]

# Commit with descriptive message
git commit -m "feat: [summary of improvements]

Changes from user session:
- [change 1]
- [change 2]
- [change 3]

These improvements were discovered during real usage and verified working."

# Push branch
git push -u origin "$BRANCH_NAME"

# Create PR
gh pr create \
  --title "[summary]" \
  --body "$(cat <<'EOF'
## What Changed

[Bullet list of changes]

## Why

[What problem was encountered during usage]

## How It Was Verified

[How the fix was tested and confirmed working]

## Files Changed

[List of files]

---
Auto-generated by Goal Agent `/goal:contribute` from a real usage session.
EOF
)"
```

## Step 5: Cleanup

```bash
# Switch back to main
git checkout main

# Report PR URL to user
echo "PR created: [URL]"
```

## For Non-Owner Users (Future)

When the plugin is used by someone who is NOT the repo owner:
1. Fork the repo first: `gh repo fork ishaquehassan/goal-agent --clone=false`
2. Add fork as remote: `git remote add fork https://github.com/USERNAME/goal-agent.git`
3. Push to fork: `git push fork $BRANCH_NAME`
4. Create PR from fork: `gh pr create --repo ishaquehassan/goal-agent --head USERNAME:$BRANCH_NAME`

This ensures any user can contribute, not just the repo owner.
